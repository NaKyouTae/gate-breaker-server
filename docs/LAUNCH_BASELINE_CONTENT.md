# Gate Breaker - 운영 첫 배포 기본 콘텐츠 명세

## 1. 목적

운영 첫 배포 시 반드시 있어야 하는 초기 콘텐츠(던전/몬스터/아이템/드랍/도감/설정)를 정의한다.
기준 소스는 `prisma/seed.ts` 및 `src/codex/codex.service.ts`이다.

## 2. 초기 콘텐츠 총량

- 던전: 5개
- 몬스터: 11개
- 아이템: 13개
- 드랍 테이블 행: 35개
- 게임 설정(GameConfig): 6개 카테고리 기본값

## 6. 드랍 테이블(필수)

드랍 테이블은 아래 기준으로 시드에 포함되어야 한다.

- 슬라임 계열: 점액/포션/입문 장비/강화석
- 고블린 계열: 이빨/포션/철 장비/강화석
- 오크 계열: 철~미스릴 장비/강화석/고급 포션
- 언데드 계열: 미스릴 장비/강화석/고급 포션
- 드래곤 계열: 용의 검/미스릴 갑옷/강화석

정확한 드랍 확률 값은 `prisma/seed.ts`를 기준으로 유지한다.

## 7. 도감(필수 동작)

도감은 별도 테이블 없이 운영된다.

- 아이템 도감 해금 기준: 유저 인벤토리에 해당 아이템이 존재하면 `discovered=true`
- 몬스터 도감 조우 기준: 유저가 해당 몬스터 던전 전투 로그를 가지면 `encountered=true`

즉, 첫 배포 시 필요한 것은 **아이템/몬스터/던전/드랍 시드 데이터**이며, 도감 데이터는 런타임 계산된다.

## 8. 게임 설정(GameConfig) 기본값

초기 배포 시 반드시 `seedDefaults()` 또는 시드에서 아래 카테고리 값이 채워져야 한다.

- `enhance` (강화 확률/페널티/비용)
- `battle` (데미지 공식, 패배 페널티, 도망 확률 설정값)
- `dungeon` (입장 쿨다운/일일 횟수 설정값)
- `drop` (등급 가중치/보스 보너스)
- `levelup` (경험치 공식/성장치)
- `shop` (판매 비율/갱신 주기)

## 9. 첫 배포 실행 순서

1. DB 마이그레이션 적용
2. 시드 실행(`pnpm prisma db seed`)
3. 관리자 계정 생성/권한 확인
4. 관리자에서 아이템/몬스터 이미지 업로드
5. 앱에서 도감, 던전 입장, 전투 드랍, 상점 구매 동작 점검
