# Weapon-Only System (무기 전용 운영안)

## 목적
- 방어구/장신구 확장 없이 `무기`만으로 성장 루프를 단순화한다.
- 강화는 최대 `+20`까지 운영한다.
- 재배포 시 무기 기준 데이터가 자동 삽입되도록 한다.

## 등급 정책
- 모든 무기의 등급(rarity)은 `COMMON`으로 통일한다.
- 무기의 차별화는 **강화 수**로만 이루어진다.
- 등급 기반 이름 색상 구분은 사용하지 않는다.

## 가격 정책
- 모든 무기의 상점 구매가(buyPrice)는 `0` (무료)이다.
- 모든 무기의 판매가(sellPrice)는 `0`이다.
- 무기는 누구나 무조건 구매할 수 있으며, 강화 시 파괴 리스크가 존재한다.

## 기본 스탯 정책
- 모든 무기의 기본 공격력(baseAttack)은 `20`으로 통일한다.
- 공격력 차이는 강화 수에 의해서만 발생한다.

## 네이밍 정책
- 추천 정책: `기본 이름 고정 + 강화 수치 표기`
- 예시: `파수 장검 +17`

### 왜 이 정책이 좋은가
- 강화마다 이름 변경보다 운영/밸런싱/DB 관리가 단순하다.
- 유저가 장비를 직관적으로 인지한다.
- 채팅/로그/거래/랭킹 등 모든 뷰에서 표기가 일관된다.

## 강화 칭호(연출용)
- 실데이터 이름은 바꾸지 않고, UI에서만 접두어를 덧붙인다.
- 5강 단위 구간 칭호를 사용한다.

| 강화 구간 | 칭호 |
|---|---|
| +0 ~ +4 | 기본 |
| +5 ~ +9 | 예리한 |
| +10 ~ +14 | 균열의 |
| +15 ~ +19 | 심연의 |
| +20 | 전설의 |

표기 예시
- 실제 DB 이름: `파수 장검`
- UI 표시: `심연의 파수 장검 +17`

## 강화 수 기반 색상 체계
- 무기 이름/테두리/글로우 색상은 **강화 수**에 따라 변한다.
- 강화 칭호 구간과 동일한 기준을 사용한다.

| 강화 구간 | 색상 | 색상 코드 | 글로우 |
|---|---|---|---|
| +0 ~ +4 | 회색 (기본) | `#ddd` | `rgba(200,200,200,0.3)` |
| +5 ~ +9 | 파랑 (예리한) | `#4a9eff` | `rgba(74,158,255,0.4)` |
| +10 ~ +14 | 보라 (균열의) | `#b048f8` | `rgba(176,72,248,0.4)` |
| +15 ~ +19 | 주황 (심연의) | `#ff8c00` | `rgba(255,140,0,0.5)` |
| +20 | 빨강 (전설의) | `#ff2d55` | `rgba(255,45,85,0.5)` |

### 적용 범위
- 인벤토리: 아이템 이름, 테두리, 장착 글로우
- 강화 화면: 아이템 이름, 아이콘 글로우
- 대시보드: 장착 무기 글로우
- 채널: 강화 목록 아이템 이름/테두리

### 구현 위치
- 공통 유틸: `apps/app/src/lib/enhance-color.ts`
- `getEnhanceColor(enhanceLevel)` → `{ color, glow }` 반환

## 기본 무기 카탈로그(8종)
- 균열 단검 (ATK 20, 0G)
- 파수 장검 (ATK 20, 0G)
- 용광 도끼 (ATK 20, 0G)
- 월광 창 (ATK 20, 0G)
- 폭풍 활 (ATK 20, 0G)
- 심연 지팡이 (ATK 20, 0G)
- 낙뢰 권갑 (ATK 20, 0G)
- 성흔 대검 (ATK 20, 0G)

## DB 자동 삽입 방식
- 파일: `src/bootstrap/weapon-catalog.seed.ts`
- 서버 시작 시 `PrismaService.onModuleInit()`에서 실행한다.
- 동작 방식
  - `무기 외 장비(ARMOR/GLOVE/SHOE/RING/NECKLACE)`를 먼저 삭제
  - 삭제 시 연관 `drop_table`, `inventory`도 함께 정리
  - 무기 이름 기준으로 기존 아이템 조회
  - 있으면 업데이트, 없으면 생성 (idempotent)
  - 카테고리는 `무기`, 타입은 `WEAPON`으로 강제

## 데이터 스크립트 반영
- `prisma/seed.ts` 실행 시에도 동일하게 무기 외 장비를 정리한다.
- `src/bootstrap/launch-bootstrap.ts` 실행 시에도 동일하게 무기 외 장비를 정리한다.

## 재배포 체크리스트
1. 서버 배포 후 애플리케이션 기동
2. 로그에서 weapon catalog seed 성공 확인
3. 어드민 아이템 목록에서 무기 8종 확인
4. 강화 +20까지 정상 동작 확인
5. 강화 수별 색상 변경 확인

## 비고
- 기존 방어구/소모품/재료 데이터를 자동 삭제하지는 않는다.
- 완전 무기 전용 초기화를 원하면 별도 정리 SQL/관리자 기능으로 일괄 정리한다.
