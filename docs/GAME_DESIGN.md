# Gate Breaker - 서버 기준 게임 디자인 문서

## 문서 정보

- 기준 저장소: `gate-breaker-server`
- 기준일: 2026-03-13
- 기준 범위: Prisma 스키마, Seed 데이터, 서비스 로직, GameConfig 기본값
- 목적: "기획 의도"보다 "현재 서버 구현"을 우선 반영한 운영 문서

## 1. 게임 코어 루프 (현재 구현)

```text
던전 선택
→ 던전 입장 (`POST /dungeon/:id/enter`)
→ 랜덤 몬스터 1:1 전투
→ 행동 선택 (`attack` / `skill` / `item` / `escape`)
→ 승리 시 보상(경험치, 골드, 드랍) 또는 패배 시 페널티
→ 인벤토리 정리(장착/판매/버리기)
→ 강화 시도
→ 상위 던전 도전
```

## 2. 플레이어/전투 스탯

### 기본 스탯

| 스탯 | 기본값 | 설명 |
|---|---:|---|
| 레벨 | 1 | 경험치 누적 시 상승 |
| HP / Max HP | 100 / 100 | 전투 체력 |
| MP / Max MP | 50 / 50 | 스킬 자원 |
| 공격력 | 10 | 기본 공격 계수 |
| 방어력 | 5 | 피해 감소 |
| 치명타율 | 0.05 | 5% |

### 레벨업 성장 (기본 설정)

- 필요 경험치 공식: `floor(base * level^1.5)`, `base = 100`
- 레벨업당 증가치:
- HP +10
- MP +5
- 공격력 +2
- 방어력 +1

## 3. 장비/아이템 시스템

### 아이템 타입

- 장비: `WEAPON`, `ARMOR`, `GLOVE`, `SHOE`, `RING`, `NECKLACE`
- 비장비: `MATERIAL`, `CONSUMABLE`

### 장착 슬롯 규칙

- 슬롯 6종(무기/갑옷/장갑/신발/반지/목걸이)
- 같은 슬롯 재장착 시 기존 장비 자동 해제

### 수량 처리 규칙

- `MATERIAL`, `CONSUMABLE`은 스택(수량 증가)
- 장비류는 개별 인벤토리 엔트리 생성

### 등급 체계

- 무기: 모든 무기는 `COMMON` 등급으로 통일. 차별화는 **강화 수**로만 이루어진다.
- 비무기: `COMMON`, `RARE`, `EPIC`, `LEGENDARY`, `MYTHIC` 등급 유지
- 기본 드랍 가중치(설정값):
- Common 0.60
- Rare 0.25
- Epic 0.10
- Legendary 0.04
- Mythic 0.01

### 무기 강화 수 기반 색상

- 무기 이름/테두리 색상은 강화 수에 따라 변한다.
- +0~+4: 회색(#ddd) / +5~+9: 파랑(#4a9eff) / +10~+14: 보라(#b048f8) / +15~+19: 주황(#ff8c00) / +20: 빨강(#ff2d55)

## 4. 강화 시스템 (현재 기본값)

### 강화 범위

- `+0`부터 `+20`까지
- 비용: `floor(100 * 1.5^현재강화레벨)`
- 실패 페널티 유형: `none`, `downgrade_1`, `downgrade_2`, `destroy`

### 강화 확률 상세

| 강화 | 성공률 | 실패 페널티 |
|---|---:|---|
| +0 → +1 | 100% | 유지 (`none`) |
| +1 → +2 | 95% | 유지 (`none`) |
| +2 → +3 | 90% | 유지 (`none`) |
| +3 → +4 | 85% | 유지 (`none`) |
| +4 → +5 | 75% | 유지 (`none`) |
| +5 → +6 | 65% | 유지 (`none`) |
| +6 → +7 | 55% | 유지 (`none`) |
| +7 → +8 | 45% | 유지 (`none`) |
| +8 → +9 | 40% | 유지 (`none`) |
| +9 → +10 | 35% | 유지 (`none`) |
| +10 → +11 | 30% | 1단계 하락 (`downgrade_1`) |
| +11 → +12 | 25% | 1단계 하락 (`downgrade_1`) |
| +12 → +13 | 20% | 1단계 하락 (`downgrade_1`) |
| +13 → +14 | 17% | 1단계 하락 (`downgrade_1`) |
| +14 → +15 | 15% | 1단계 하락 (`downgrade_1`) |

| +19 → +20 | 3% | 파괴 (`destroy`) |

### 장비 능력치 반영

- 전투 진입 시 장비 능력치 계산식:
- `장비 반영치 = floor(baseStat * (1 + 강화레벨 * 0.1))`

## 5. 전투 시스템 (현재 구현)

### 행동

- `POST /battle/attack`
- `POST /battle/skill` (MP 10 소모, 공격력 1.5배 기반)
- `POST /battle/item` (소비 아이템 1개 사용, 현재 고정 30 회복)
- `POST /battle/escape` (현재 고정 50% 성공)

### 데미지 공식

```text
기본 피해 = 공격력 - 방어력 + random(-3, +3)
치명타 시 = floor(기본 피해 * 1.5)
최소 피해 = 1
```

### 전투 결과

- 승리: 몬스터 보상(경험치, 골드, 드랍)
- 패배: 보유 골드의 10% 손실
- 도망: 보상 없음, 전투 로그 기록

## 7. 경제 시스템 (현재 구현)

### 획득

- 전투 승리 골드
- 아이템 판매 골드

### 소모

- 상점 구매 (`/shop/buy`)
- 강화 비용 (`/enhance`)
- 패배 페널티(골드 10%)

### 상점

- 구매 가능 조건: `buyPrice >= 0` (buyPrice가 null이 아닌 모든 아이템)
- 무기: 모두 무료(0G). 강화 시 파괴 리스크가 비용을 대체한다.
- 판매: 인벤토리 기준 `sellPrice * 판매수량`

## 8. 운영/밸런스 조정 포인트

`/config` API로 주요 수치를 런타임 조정 가능:

- `enhance/*`: 강화 확률/패널티/비용 계수
- `battle/damage_formula`, `battle/defeat_penalty`
- `drop/rarity_weights`, `drop/boss_drop_bonus`
- `levelup/exp_table`, `levelup/stat_growth`
- `shop/sell_rate`, `shop/refresh_interval`

- `dungeon.cooldown`, `dungeon.max_daily_entries` 설정값은 존재하지만 입장 제한 로직은 미연동 상태
